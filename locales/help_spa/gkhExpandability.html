Ampliabilidad

Existen dos tipos de extensiones disponibles para GEDKeeper: scripts Lua y plugins escritos en C# o VB.NET.

Las siguientes son las principales razones para ampliar una aplicación con unidades plugin:

  • Añadir nuevas funcionalidades a la aplicación sin tener que reconstruir el código fuente y desplegar una nueva versión.
  • Añadir nuevas funcionalidades a la aplicación sin disponer del código fuente necesario.

Para crear plugins, debe utilizar la biblioteca común GKCore del núcleo de la aplicación.

Los plugins pueden incluir tanto extensiones totalmente independientes como widgets auxiliares para la gestión de datos e informes.
